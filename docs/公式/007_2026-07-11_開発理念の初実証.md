# 開発理念の初実証

日付: 2026-07-11

## 背景

GUI の DEBUG 欄を折りたたみ表示へ変更した。

当初、この変更が原因で画面表示に問題があるように見えたため、
一度 Revert を実施した。

しかし、その後の検証により、原因は UI の変更ではなく、
起動方法の違い（実行環境）による既知の問題であることが分かった。

## 検証

「実用版」と「開発版」を比較した。

実用版
・ham_control の 13 番ボタンから起動する運用中バイナリ

開発版
・現在開発中のソースコードから生成したバイナリ

両者を同じ条件で実機確認した結果、

・DEBUG 折りたたみ表示
・GUI 表示
・送受信動作

すべて同じ結果になることを確認した。

Revert は取り消され、
変更は正式な実装として採用された。

## 得られたこと

これは、

「道は違えど、結果は同じ。」

という開発理念が、

実際の開発手順として機能した最初の事例となった。

今後も、

・推測では判断しない
・実用版と比較する
・実機で確認する
・Git に記録を残す

という手順を開発の基本とする。
