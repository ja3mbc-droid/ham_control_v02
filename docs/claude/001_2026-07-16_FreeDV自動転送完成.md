# 008_2026-07-16 FreeDV自動転送機能 完成記録

## 概要
FreeDV → UDP(port 2237, WSJT-X NetworkMessageプロトコル) → ham_control_v02 → HAMLOG(CSV)
の自動転送経路が、GUI表示まで含めて完全に動作するようになった。

## 発見・修正したバグ

### 1. QDateTimeのフィールド長誤り
日付(Julian Day)を4バイト(u32)と誤認識していたが、実際は8バイト(u64)。
tcpdumpの実データを手計算で解析し、ユリウス日番号(2461236 = 2026年7月頃)と一致することで特定。

### 2. 文字列エンコーディングの誤り
QStringをUTF-16BE(2byte/文字)と誤認識していたが、実際はUTF-8(1byte/文字)。

### 3. クライアントID文字列の読み飛ばし漏れ
ペイロード先頭にQString形式のクライアントID("FreeDV")が入っており、
これを読み飛ばさずに直接date_time_offから読もうとしていたためオフセットがずれていた。

### 4. QsoLogged構造体のフィールド抜け
tx_power(report_receivedとcommentsの間)、op_call(date_time_onとmy_callの間)の
2フィールドが構造体定義から抜けており、それ以降の全フィールドがズレて読まれていた。

### 5. 日時フォーマットの改善
Julian Day+msecの生値ではなく、"YYYY-MM-DD HH:MM:SS UTC"形式に変換して
既存CSVフォーマットと統一。

### 6. GUI CALL欄が更新されないバグ
FreeDvLogAdapter::from_qso()が常にstatus: Noneを設定していたため、
ui.rs側の「ALL.TXTから読込」ボタン処理でNoneの分岐に入り、
CALL欄等への反映処理が一切実行されていなかった。
Confirm Log OKの時点で交信は完結しているため、status: Some(QsoStatus::Complete)
に修正して解決。

## アーキテクチャ変更(006の実装)

- Receiver(wsjtx_receiver.rs): UDP受信・パースのみ
- Parser(wsjtx_protocol.rs): NetworkMessageバイナリプロトコルのパース
- Adapter(freedv_log.rs): QsoLogged → QsoRecord変換、LogAdapter trait実装
  - Mutex<Option<QsoRecord>>で最新QSOを保持し、latest_qso()で取得可能に
- LogManager(log_manager.rs): FreeDvLogAdapterの唯一の所有者
  - handle_freedv_qso()でCSV書き込みとGUI用の最新値保持を両方実施
  - latest_qso()はFreeDVを最優先でチェック

## 実機確認結果
複数のFreeDV交信(JH6DUL, JE6HKP, JA6KJQ, JA3MMH等)が
CSV(~/ham_control_v02_activity.csv)へ正しく記録され、
GUI画面のCALL/RST SENT/TIME_ON/TIME_OFF/QSO MODE欄にも
正しく反映されることを確認した。
